## Overview

Recur uses [MMKV](https://github.com/mrousavy/react-native-mmkv) for local data persistence. MMKV is an ultra-fast, encrypted key-value storage library that's **up to 30x faster than AsyncStorage**.

## Implementation

### MMKVClient

Recur wraps MMKV in a simple client for JSON serialization and error handling.

From clients/mmkv.ts:7:

```typescript
import { createMMKV, MMKV } from 'react-native-mmkv';
import { Logger } from './logger';

const logger = new Logger('MMKV Client');

export class MMKVClient {
  private storage: MMKV;

  constructor() {
    this.storage = createMMKV();
  }

  set<T>(key: string, value: T) {
    this.storage.set(key, JSON.stringify(value));
  }

  get<T>(key: string): T | null {
    const value = this.storage.getString(key);
    if (!value) {
      return null;
    }

    try {
      return JSON.parse(value);
    } catch (error) {
      logger.error('Failed to parse MMKV value for key: ' + key, error);
      return null;
    }
  }

  delete(key: string) {
    this.storage.remove(key);
  }
}
```

## Data Integrity

### Validation on Read

All data is validated when read from storage using Zod schemas.

From actions/subscription.ts:106:

```typescript
export const getSubscriptions = () => {
  const stored = mmkv.get<Subscription[]>(SUBSCRIPTIONS_KEY);
  const normalized = normalizeSubscriptions(stored);
  
  // Auto-repair corrupted data
  if (Array.isArray(stored) && normalized.length !== stored.length) {
    mmkv.set(SUBSCRIPTIONS_KEY, normalized);
  }
  
  return normalized;
};
```

### Graceful Degradation

From actions/subscription.ts:17:

```typescript
const normalizeSubscriptions = (raw: unknown): Subscription[] => {
  if (!Array.isArray(raw)) return [];

  const parsedArray = Zod_SubscriptionArray.safeParse(raw);
  if (parsedArray.success) {
    return parsedArray.data;
  }

  // Recover valid items from corrupted array
  const cleaned: Subscription[] = [];
  let dropped = 0;
  
  for (const item of raw) {
    const parsed = Zod_Subscription.safeParse(item);
    if (parsed.success) {
      cleaned.push(parsed.data);
    } else {
      dropped += 1;
    }
  }

  if (dropped > 0) {
    logger.warn(`Dropped ${dropped} invalid subscription(s) from storage`);
  }

  return cleaned;
};
```

This ensures the app **never crashes** due to corrupted storage data.

## Real-Time Updates

### Subscription Pattern

Recur implements a simple pub/sub pattern for reactive updates.

From actions/user.ts:42:

```typescript
type OnboardingListener = (isComplete: boolean) => void;
const onboardingListeners = new Set<OnboardingListener>();

export const subscribeToOnboarding = (listener: OnboardingListener) => {
  onboardingListeners.add(listener);
  return () => {
    onboardingListeners.delete(listener);
  };
};

export const setOnboardingComplete = () => {
  mmkv.set(ONBOARDING_COMPLETE_KEY, true);
  // Notify all listeners
  onboardingListeners.forEach((listener) => listener(true));
};
```

## Performance Characteristics

### Synchronous Operations

MMKV operations are **synchronous**, meaning you can read and write immediately:

```typescript
// No async/await needed!
mmkv.set('key', value);
const data = mmkv.get('key');
console.log(data); // Available immediately
```

- Efficient memory usage
- Fast startup times
- No data loss on app crashes

## Best Practices

### Always Validate

```typescript
import { Zod_Subscription } from '@/types';

const stored = mmkv.get<Subscription[]>('subscriptions');
const validated = Zod_Subscription.array().safeParse(stored);

if (validated.success) {
  return validated.data;
} else {
  logger.error('Invalid stored data', validated.error);
  return [];
}
```

### Use Type Parameters

```typescript
// Good: Type-safe
const subscriptions = mmkv.get<Subscription[]>('subscriptions');

// Bad: Type inference fails
const subscriptions = mmkv.get('subscriptions');
```

### Handle Nulls

```typescript
const userName = mmkv.get<string>('user_name');

if (!userName) {
  // First time user - no name set yet
  return 'Guest';
}

return userName;
```

### Batch Updates

```typescript
// Instead of multiple writes:
mmkv.set('subscription_1', sub1);
mmkv.set('subscription_2', sub2);
mmkv.set('subscription_3', sub3);

// Store as array:
mmkv.set('subscriptions', [sub1, sub2, sub3]);
```

## Migration Support

If you need to migrate from AsyncStorage:

```typescript
import AsyncStorage from '@react-native-async-storage/async-storage';
import { mmkv } from '@/integrations/mmkv';

async function migrateFromAsyncStorage() {
  const keys = await AsyncStorage.getAllKeys();
  
  for (const key of keys) {
    const value = await AsyncStorage.getItem(key);
    if (value) {
      mmkv.set(key, JSON.parse(value));
    }
  }
  
  // Clear old storage
  await AsyncStorage.clear();
}
```
