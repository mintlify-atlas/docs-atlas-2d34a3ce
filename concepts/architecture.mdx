## Architecture Patterns

### File-Based Routing

Recur uses Expo Router's file-based routing system:

```tsx
app/
├── _layout.tsx          // Root layout with theme & navigation
├── (tabs)/              // Tab-based navigation group
│   ├── index.tsx        // Home screen
│   └── settings/        // Settings stack
└── onboarding/          // Onboarding flow
```

### State Management

Recur uses a lightweight, action-based pattern with MMKV storage:

```typescript
// Action modules in /actions
- subscription.ts   // CRUD operations for subscriptions
- list.ts          // Manage subscription lists
- currency.ts      // Currency preferences
- user.ts          // User settings & onboarding state
```

See app/_layout.tsx:30 for the state management pattern using React's `useReducer`.

### Data Validation

All data is validated using Zod schemas before storage:

```typescript
// From types.ts
export const Zod_Subscription = z.object({
  id: z.string(),
  name: z.string().trim().min(1),
  icon: z.string().nullable(),
  listId: Zod_List.shape.id.trim().min(1).nullable().optional(),
  frequency: Zod_SubscriptionFrequency,
  amount: z.number().nonnegative(),
  currency: Zod_Currency,
  startDate: z.coerce.date(),
});
```

### Component Structure

```
components/
├── ui/              // Reusable UI primitives
├── subscription/    // Subscription-specific components
└── layout/          // Layout components
```

### Client Services

Utility clients for cross-cutting concerns:

```typescript
clients/
├── mmkv.ts      // Storage client wrapper
└── logger.ts    // Logging utility
```

## Type Safety

Recur maintains strict type safety throughout:

1. **Schema-first design**: All data models defined with Zod schemas
2. **Type inference**: TypeScript types inferred from Zod schemas using `z.infer<>`
3. **Runtime validation**: All external data validated at runtime
4. **TypeScript strict mode**: Full type checking enabled

## Performance Optimizations

### MMKV Storage

MMKV provides synchronous, ultra-fast storage:
- **30x faster** than AsyncStorage
- Encrypted and secure
- Synchronous API for instant reads
- Lazy loading and caching

See clients/mmkv.ts:1 for the storage implementation.

### Native Animations

Reanimated 4 runs animations on the native thread:
- 60 FPS animations without JS thread blocking
- Gesture-driven interactions
- Smooth drag-and-drop lists

### Code Splitting

Expo Router enables automatic code splitting per route, reducing initial bundle size.
