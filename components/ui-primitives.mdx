# UI Primitives

UI primitives are the foundational components of Recur's design system. Located in `components/ui/`, these components provide consistent styling, behavior, and accessibility across the app.

## Overview

All UI primitives use **NativeWind** for styling and follow these principles:

- **Variant-based design** using `class-variance-authority`
- **Platform-aware** with web-specific optimizations
- **Theme support** for light/dark modes
- **Fully typed** with TypeScript
- **Accessible** with proper roles and labels

## Layout Components

### Sheet

A bottom sheet component for mobile-friendly dialogs.

**File:** `components/ui/sheet.tsx`

```tsx
import { Sheet, SheetTrigger, SheetContent } from '@/components/ui/sheet';

function Example() {
  const [open, setOpen] = React.useState(false);

  return (
    <Sheet open={open} onOpenChange={setOpen}>
      <SheetTrigger asChild>
        <Pressable>
          <Text>Open Sheet</Text>
        </Pressable>
      </SheetTrigger>
      <SheetContent
        title="Sheet Title"
        description="Sheet description"
        height="90%"
        scrollable={true}>
        {/* Content */}
      </SheetContent>
    </Sheet>
  );
}
```

**Props:**

<ParamField path="title" type="string" optional>
  Sheet header title.
</ParamField>

<ParamField path="description" type="string" optional>
  Sheet header description.
</ParamField>

<ParamField path="height" type="DimensionValue" default="90%">
  Sheet height (percentage or absolute value).
</ParamField>

<ParamField path="scrollable" type="boolean" default={true}>
  Whether content should be scrollable.
</ParamField>

**Features:**
- Slide-in animation from bottom
- Backdrop overlay with dismiss on tap
- Handle indicator at top
- Close button in header
- Configurable height

## Form Controls

### Button

A pressable button with multiple variants and sizes.

**File:** `components/ui/button.tsx`

```tsx
import { Button } from '@/components/ui/button';
import { Text } from '@/components/ui/text';

<Button variant="default" size="lg" onPress={handlePress}>
  <Text>Click Me</Text>
</Button>
```

**Variants:**

<CodeGroup>
```tsx default
<Button variant="default">
  <Text>Primary Action</Text>
</Button>
```

```tsx outline
<Button variant="outline">
  <Text>Secondary Action</Text>
</Button>
```

```tsx ghost
<Button variant="ghost">
  <Text>Subtle Action</Text>
</Button>
```

```tsx destructive
<Button variant="destructive">
  <Text>Delete</Text>
</Button>
```

```tsx link
<Button variant="link">
  <Text>Link Style</Text>
</Button>
```
</CodeGroup>

**Sizes:**

```tsx
<Button size="sm">Small</Button>
<Button size="default">Default</Button>
<Button size="lg">Large</Button>
<Button size="icon">{/* Icon only */}</Button>
```

**Disabled State:**

```tsx
<Button disabled={true}>
  <Text>Disabled</Text>
</Button>
```

**Implementation Details:**

```tsx
const buttonVariants = cva(
  cn(
    'group shrink-0 flex-row items-center justify-center gap-2 rounded-md shadow-none',
    Platform.select({ /* ... */ })
  ),
  {
    variants: {
      variant: {
        default: cn(
          'bg-primary active:bg-primary/90 shadow-sm shadow-black/5',
          Platform.select({ web: 'hover:bg-primary/90' })
        ),
        // ... other variants
      },
      size: {
        default: cn('h-10 px-4 py-2 sm:h-9'),
        // ... other sizes
      },
    },
  }
);
```

### Input

A text input component with consistent styling.

**File:** `components/ui/input.tsx`

```tsx
import { Input } from '@/components/ui/input';

const [value, setValue] = React.useState('');

<Input
  value={value}
  onChangeText={setValue}
  placeholder="Enter text"
  keyboardType="default"
/>
```

**Common Patterns:**

<CodeGroup>
```tsx text-input
<Input
  placeholder="Name"
  value={name}
  onChangeText={setName}
  autoCapitalize="words"
/>
```

```tsx numeric-input
<Input
  placeholder="0.00"
  keyboardType="numeric"
  inputMode="decimal"
  value={amount}
  onChangeText={setAmount}
/>
```

```tsx with-icon
<View className="relative justify-center">
  <Text className="absolute left-3 z-10">$</Text>
  <Input className="pl-7" />
</View>
```
</CodeGroup>

**Features:**
- Automatic debouncing for controlled values
- Platform-specific styling (web vs native)
- Focus ring on web
- Invalid state styling with `aria-invalid`
- Shadow effects

## Best Practices

### Always Use UI Primitives

Prefer UI primitives over raw React Native components:

```tsx
// ✅ Good
import { Button } from '@/components/ui/button';
import { Text } from '@/components/ui/text';

<Button onPress={handlePress}>
  <Text>Action</Text>
</Button>

// ❌ Avoid
import { Pressable, Text as RNText } from 'react-native';

<Pressable onPress={handlePress}>
  <RNText>Action</RNText>
</Pressable>
```

### Compose Components

Build complex UIs by composing primitives:

```tsx
<Card>
  <CardHeader>
    <CardTitle>Subscription Details</CardTitle>
  </CardHeader>
  <CardContent>
    <View className="gap-4">
      <Input placeholder="Name" />
      <Button variant="default">
        <Text>Save</Text>
      </Button>
    </View>
  </CardContent>
</Card>
```

### Use Variants

Leverage built-in variants instead of custom styles:

```tsx
// ✅ Good
<Button variant="outline" size="sm">
  <Text>Action</Text>
</Button>

// ❌ Avoid
<Button className="border border-gray-300 h-8 px-2">
  <Text>Action</Text>
</Button>
```
